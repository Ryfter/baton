## Theme

rc1 stood up the **Fleet Conductor** evolution (capability routing + cost engine + legibility
dashboard). rc2 makes it **economic and autonomous**: a governed-fleet pipeline (Triage → Usage
→ Projects → Research Gate) that classifies, gates, and routes work cost-optimally — capped by
the **Conductor (`/baton:go`)**, the natural-language front door that plans then executes a task
DAG autonomously, interrupting only for a budget cap or a destructive action. North star
unchanged: **autonomy** (stop the per-step 1/2 prompts) + **legibility** (always show, in plain
English, what each agent is doing) — with **cost-optimization** as the through-line.

## Highlights

- **The Conductor (`/baton:go`)** — the capstone. A natural-language front door: plan-then-execute
  over a task DAG, full-auto, interrupting only on a **budget cap** or a **destructive action**,
  with an event ledger + decision (autonomous-guess) ledger + plain-English report as the
  legibility safety net. Style-A substrate today with a file-based Style-B seam for a future
  standalone broker + cockpit.
- **Governed-fleet v2 pipeline (Sprints 1–4)** — Triage classifies, the Usage Governor gates,
  Projects sync writes board state, and the Research Gate issues a build/adopt/adapt verdict
  before non-trivial work. All four ride the same cost-aware routing spine.
- **Models-as-tools registry** — `/baton:models` inventory engine (probes, tags, recommendations),
  capability claims + context floors, economy/champion selection, and gauntlet-scorecard import
  into the quality blend.
- **Cost-Optimization Engine Slices B + C** — the draft→finish cascade, then the cascade wired
  into the autonomous run-loop (concurrent driver, paid-peak gate, surge-scaled parallelism).

## What shipped (since v1.2.0-rc1)

### The Conductor — `/baton:go` (merged `6e72c9b`, PR #59)
The natural-language front door. `conductor-lib.ps1` parses a plan into a task DAG (Kahn
topological order), walks it under **exactly two interrupt guards** — a budget cap and a
`reversible:false` destructive action, both checked **before** each spawn (structural
guard-before-spawn) — and emits four files under `$BATON_HOME/runs/<run-id>/`: `plan.json`,
`events.jsonl`, `decisions.jsonl` (the autonomous-guess ledger), and a plain-English `report.md`.
Ambiguity/forks/under-cap spend are auto-guessed and logged, never prompted. `fleet-go.ps1` is the
CLI; `/baton:go` the command. Box-private (`budget_cap` null in shared code); hermetic suite
**59/59** (incl. a zero-network child-process CLI test). Final whole-branch adversarial review:
READY TO MERGE, no Critical defects. Decisions: substrate Style-A + Style-B seam; full-auto with a
minimal two-interrupt set; file-based hand-off contract.

### Governed-fleet v2 pipeline — Sprints 1–4
- **Sprint 1 — Triage Agent** (merged PR #52). `Invoke-TriageAgent` reads url/file/text (exactly
  one source), builds a strict-JSON-schema prompt, dispatches via the governed fleet, parses with
  fallback, and escalates on low confidence. `/baton:triage` + `fleet-triage.ps1`.
- **Sprint 2 — Usage Governor** (merged PR #54). Rank-gates dispatch against usage/budget state so
  premium spend is governed at the pipeline edge.
- **Sprint 3 — GitHub Projects sync** (merged PR #56). Labels classify, Project fields decide;
  pure mapping/planning layer + `gh` I/O behind a `$GhInvoker` seam, dry-run by default. Decision
  d049.
- **Sprint 4 — Research Gate** (merged PR #58). A build/adopt/adapt verdict memo before non-trivial
  work — `research-gate-lib.ps1` (pure layer + seamed evidence search/orchestration),
  `/baton:research-gate`, evidence-grounded on a cheap model. Prevents Doc2MD-style wasted effort.

### Models-as-tools registry (decisions d043/d044)
`/baton:models` inventory engine — probes providers, tags by class, recommends, and surfaces a
scorecard import. Capability **claims** + context **floors** drive economy/champion selection;
the judge is resolved by capability claim with a first-local fallback; `Import-GauntletScorecard`
feeds a gauntlet evidence bucket into the quality blend. LM Studio 0.3.x native shape normalized;
explicit per-box pins (no `auto` pseudo-pins), one-model-server-per-box honored.

### Cost-Optimization Engine — Slices B + C
- **Slice B — draft→finish cascade** (merged PR #44). A two-stage cascade that drafts cheap then
  finishes, salvaging the best usable draft on defer/escalate. `/baton:route --cascade`.
- **Slice C — cascade in the run-loop** (merged PR #46). The cascade wired into the autonomous
  driver: concurrent execution with effective-rank ordering, a paid-peak gate, and surge-scaled
  `MaxParallel`; `run-backlog` passthrough.

### Fixes + hardening
Bootstrap manifest gaps closed (backlog drivers were stale); flat slash-command refs namespaced
to `/baton:*`; runs-root test isolation; LM Studio model-pin + cold-load timeout handling.
