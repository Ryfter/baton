# v1.15.0 — Direct-model commands + token telemetry (a direct line to every instrument)

**Date:** 2026-07-11 · **PR:** #86 (squash `9b73687`) · **Track:** multi-model conductor (#2 in Kevin's 2026-07-11 order)

Talking to one specific fleet model from inside Claude Code used to mean prose-routing
through the conductor or dropping to a raw terminal. **`/baton:codex`, `/baton:grok`,
`/baton:gemini` (+ `/baton:agy`)** are now direct lines — and unlike a raw CLI call,
every dispatch is journaled and Usage-Governor-metered through Baton's hardened prompt
transport. The journal also finally counts what matters: **tokens, not just calls.**

## What shipped

- **`scripts/fleet-ask.ps1`** — the one shared runner behind all four commands.
  `-Prompt` inline or `-PromptFile` (the 965-byte escape hatch); prints the model's
  stdout + an ASCII footer `-- <provider> | <N>s | exit:<code> | tok:<n>(<basis>)`;
  exit 2 with a friendly stderr line on unknown/disabled provider, unknown tier
  (lists the valid ones), or missing prompt. Namespace **A3**: canonical
  `/baton:*` commands ship + deploy with the plugin; bare `/codex`-style aliases
  are documented, not force-deployed.
- **Per-model token telemetry (observe-only).** `Get-FleetTokenUsage`: a per-row
  `token_usage` regex captures the CLI's own printed count → `exact` (codex seeded:
  `tokens used: 14,350`); no field / no match → honest `len/4` → `estimate` — the
  d059 rule: an estimate is never labelled metered. Threaded through both dispatch
  kinds into the `Invoke-Fleet` return (`tokens`, `tokens_basis`) and appended to
  every fleet journal line as a strictly-trailing ` | tok:N(basis)` field.
- **Named tier selection.** Flat `tier_<name>` fleet.yaml keys carry arg fragments;
  `{{tier_args}}` in the template gets the selected fragment (`--tier <name>`,
  `tier_default`, or empty). **`--tier all`** = the boundary tester — run the same
  prompt across every tier (capped at 16). Seed ships generic codex effort tiers
  (`-c model_reasoning_effort=low|medium|high`); real model-ID tiers are box-private.
- **Dashboard journal parser** peels the new `host:/tier:/tok:` trailing tags
  fail-closed (unknown tags stop the peel), keeping `job:`/`phase:` extraction
  intact — +14 regression tests.
- **Bonus fix:** the long-red `test-hook.ps1` "5 pipe-separated fields" assert was a
  hermeticity leak (the hook read the real box's active job); state path now isolated.
