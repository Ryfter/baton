# v1.4.0 — The Learning Conductor release (stable)

**Tag:** `v1.4.0` · **Date:** 2026-06-25 · **Previous:** `v1.3.0` (2026-06-21), rc line: `1.4.0-rc.1` (saturation) → `1.4.0-rc.2` (acceptance-gate wiring) → `1.4.0-rc.3` (effective-cost slice 1) → `1.4.0-rc.4` (effective-cost slice 2)

The **stable v1.4.0**. Promotes the untagged `1.4.0-rc.4` candidate to final. It bundles the first
post-MVP line — three slices that make the governed fleet **learn from its own economics**: an
active saturation driver, the acceptance gate wired into the Conductor, and a quality-adjusted
effective-cost metric with a per-worker leaderboard. Both test gates are green and there are
**0 open issues**; the remaining roadmap items (below) are explicitly future work. The rc line ends here.

---

## Theme

v1.3 closed the 7-sprint MVP — Baton could route, meter, gate, and remember. **v1.4 makes the
fleet learn from what it spends.** The economic thesis was *spend intelligence like money*;
v1.4 adds the missing feedback signal: Baton now measures **what it actually paid for the quality
it actually got**, per run and per worker, and saturates its pre-paid capacity before reaching for
paid tiers. North star unchanged: **autonomy** + **legibility**, with **cost-optimization** as the
through-line — and now a learned signal pointing the through-line at the right workers.

## What's new since v1.3.0

### Active Saturation Driver (PR #64; decision d057; `1.4.0-rc.1`)
`Select-Capability` now actively **up-ranks an under-utilized, opt-in, budgeted worker** so the
cost-optimal selector spends its pre-paid/free allotment first (toward Kevin's 99.9%-utilization
target) — the symmetric inverse of the Sprint-2 route-around. A boosted candidate's effective
cost-tier rank floors at `−1` (below `local`) while it has headroom, then route-around takes over
once exhausted. Opt-in per worker (`saturate: true` + `budget`), default off → zero blast radius.
`saturation-lib.ps1` + surgical `routing-lib.ps1` extension. Decisions d-sat-1..5.

### Acceptance Gate → Conductor wiring (PR #65; decision d058; `1.4.0-rc.2`)
After a successful `/baton:go` DAG walk, an **opt-in, advisory, fail-open run-level acceptance
phase** gates the finished artifact and maps the verdict to the terminal status: `accept`/`polish`
→ `completed` (polish adds the brief), `reject` → a new `rejected` status. Opt-in via
`-GateArtifact` / `-GateDiff`; no target → phase skipped → behavior byte-for-byte identical. Adds a
5th run artifact `acceptance.json` + an `## Acceptance` report section. Decisions d-cg-1..5.

### Effective Cost — slice 1, run-level (PR #66 + fix #67; decision d059, d-ec-1..5; `1.4.0-rc.3`)
**`effective_cost = realized_cost ÷ realized_quality`** — what you actually paid for the quality you
got; lower is better. When the acceptance gate produces a verdict, the Conductor writes a 6th run
artifact `effective-cost.json` + a `## Effective cost` report section; **no gate → byte-for-byte
unchanged**. The quality scalar is banded-by-verdict, refined-by-counts, floored > 0; the cost
numerator is a labelled cost-tier **estimate** behind a `-CostResolver` seam (the deployed smoke
caught that realized spend is still a `0.0` placeholder — the metric sources the budget guard's
estimate instead, so the number is honest about being a listed-price proxy). Paper-informed by
*The Price Reversal Phenomenon* (arXiv 2603.23971).

### Effective Cost — slice 2, per-worker leaderboard (PR #68; `1.4.0-rc.4`)
Folds the per-run `effective-cost.json` records into a **per-worker learned leaderboard**
(`Get-WorkerEffectiveCost`) — whole-run, single-producer-weighted, confidence-gated — and exposes
it as **`/baton:effective-cost [report] [--json] [--runs <glob>]`**, cheapest-quality-adjusted
worker first with low-confidence rows flagged tentative. Advisory / legibility only. `--json` emits
the rows array the future dashboard MODEL LEADERBOARD panel will consume.

- **Band-boundary fix (d059 deferred item 1):** the quality scalar's lower bands are now half-open
  at report precision, so a clean polish (0.6999) can never tie a pathological many-minor accept at
  0.7 — strict cross-band ordering verified by exhaustive sweep, so the leaderboard's cross-band
  ranking is sound.
- The confidence-gated `Select-Capability` re-rank (bias routing by learned effective cost) is a
  **named, deferred** future slice — advisory-first discipline, same as the saturation driver.
