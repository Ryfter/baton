# v1.16.0 ‚Äî Review Named Panel (the acceptance gate grows a roster)

**Date:** 2026-07-14 ¬∑ **Track:** feature (`scripts/gate-lib.ps1`) ¬∑ **Follows:** v1.15.1 ¬∑ **Decision:** d086

The acceptance gate could already run N independent reviewers and merge their findings, but
every reviewer was the same generic "strict reviewer." This release layers a **named panel**
on top: a config-driven roster of specialised review roles, each routed to its cheapest
capable model, each finding tagged with the role that raised it ‚Äî and, on the golden path, a
**fail-loud** posture so a degraded panel can no longer masquerade as a clean pass (the d086
point: governance on the critical path must be authoritative, not advisory theatre).

## What's new

- **Role roster** ‚Äî `references/review-roles.yaml`, a shipped seed of six generic coding
  roles: `correctness`, `security` (adversarial), `architecture`, `spec-compliance`
  (strong tier), plus `simplicity` and `framework-style` (cheap tier). Tiers map to model
  selection at runtime; the seed carries **no box-private model IDs**. Seeded into
  `$BATON_HOME` if absent, like `fleet.yaml`.
- **Role-aware prompt** ‚Äî `Build-ReviewPrompt` takes an optional `-Role`; the role identity
  (`You are the <name> reviewer. <lens>`) *replaces* the generic opening, so each reviewer
  looks through one lens instead of diluting it.
- **Panel dispatch** ‚Äî `Invoke-AcceptanceGate -Panel` picks each enabled role's model via the
  existing `Select-Capability` (constrained by the role's tier ‚Äî `strong` up to `-MaxCostTier`,
  `cheap` preferring local/free), dispatches the role-lens prompt, and records role provenance
  in each finding's `area`. One model may serve several roles. Merge / verdict / brief
  functions are **untouched**.
- **Fail-loud (`-FailLoud`, d086)** ‚Äî an explicit `--panel` with zero usable roles, or a panel
  where every reviewer was skipped, is now a **degraded** state (`degraded=$true` +
  `degraded_roles`) that surfaces loudly instead of silently accepting. Advisory default is
  unchanged when `--fail-loud` is not requested.
- **Wiring** ‚Äî `/baton:gate` and `scripts/fleet-gate.ps1` gain `--panel` and `--fail-loud`.
  With no roster and no `--panel`, behaviour is byte-for-byte the existing generic review.

## How it was built

Built almost entirely by the fleet, to conserve a near-exhausted weekly Claude budget:
**Codex implemented ‚Üí Grok reviewed (caught two real fail-loud gaps) ‚Üí Codex fixed ‚Üí**
Claude orchestrated, ran the box-private check, and verified the suite. The Codex‚ÜíGrok‚ÜíCodex
loop is the review panel performed by hand ‚Äî the pattern was used to build the pattern.
