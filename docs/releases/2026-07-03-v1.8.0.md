# v1.8.0 — "The Coach" (guided use)

**Date:** 2026-07-03 · **PR:** #76 (merge `9ba458b`) · **Decision:** d074 ·
**Spec:** `docs/superpowers/specs/2026-07-03-guided-use-coach-design.md`

Baton now suggests its own commands. Kevin's directive: features are hidden
unless suggested — fire commands at session start and suggest the right
`/baton:*` command as development goes on. One rules engine, two surfaces,
zero model cost.

## What shipped

- **Coach engine** (`scripts/coach-lib.ps1`): declarative rule table over
  locally readable state — six rules in priority order: last-run next step
  (reuses `/baton:start`'s `NextCommandMap`), gate `polish`/`reject` →
  `/baton:optimize-prompt`, promote nudge pending → `--apply`, live A/B
  verdict ready → `--pool`, budget at risk (conserve mode or ≤2-day
  forecast) → `/baton:usage`, unregistered git repo → `/baton:start`.
  Fail-open context readers (a broken signal source degrades to "no
  signal"); one-shot dedup stamps in `$BATON_HOME/coach/seen.json`; level
  knob `$BATON_HOME/coach/config.json` — `off` / `quiet` (default) /
  `teach` (adds the why).
- **Session-start digest** (`scripts/hooks/baton-coach.ps1`, registered in
  `hooks/hooks.json` after baton-init): registered projects get 3–5
  orientation lines (last-run status, prompt-pool state, budget posture,
  one suggested command); unregistered git repos get a one-shot
  "Baton available — /baton:start to onboard"; other directories get
  silence. Always exits 0; UTF-8 stdout forced; read-only except the
  onboard stamp.
- **`Next:` footers** on `/baton:usage`, `/baton:gate`, `/baton:go`,
  `/baton:optimize-prompt`: at most one suggestion at the end of
  human-readable output — one-shot per triggering state, never in
  `--json` output, never on error paths, never suggesting the command
  that just ran.

## Roadmap position

Guided use was the queued →NEXT after the GEPA optimizer arc (v1.6.0 →
v1.7.1). With the coach shipped, the optimizer loop it advertises is armed
end-to-end: the first real gate failure both feeds the optimizer AND gets
suggested to the operator. Remaining tracks: dashboard integration,
Style-B broker + cockpit, Memory Bridge, cascade routing, DORA Impact
metric — plus the coach v2 candidates (statusline glyph, `/baton:coach`
surface) if the two shipped surfaces prove the pattern.
