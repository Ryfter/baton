# v1.5.0 — The Front Porch release (stable)

**Tag:** `v1.5.0` · **Date:** 2026-07-01 · **Previous:** `v1.4.1` (learned-cost re-rank), rc line: `1.5.0-rc.1` (start slice 1) → `1.5.0-rc.2` (working-style loop) → `1.5.0-rc.3` (idea injection) → `1.5.0-rc.4` (hardening wave: slice-3 fix + CostResolver + gated GEPA optimizer)

The **stable v1.5.0**. Promotes `1.5.0-rc.4` to final. It bundles the `/baton:start` front porch
(all three slices) with two learning-loop upgrades: the realized-cost **CostResolver** (the cost
numerator moves from `estimate` toward **measured**) and the **gated GEPA prompt-optimizer pilot**
(`/baton:optimize-prompt`). All test gates are green; deployed live and smoke-verified. The rc line ends here.

---

## Theme

v1.4 made the fleet learn from what it spends. **v1.5 opens the front door and starts learning
from the *user*.** `/baton:start` is the missing legible on-ramp — a guided, teaching entry point
for someone who may not know what git is — and its slices 2/3 begin modeling *how the user works*
(interview depth, teaching level, brain-dump-anytime cadence). Underneath, the honesty through-line
advances on both ends: realized cost is now metered from real provider logs (d066), and the
Conductor's own planner prompt becomes a compiled, reflectively-improvable artifact (d069/d070).
North star unchanged: **autonomy** + **legibility**.

## What's new since v1.4.1

### `/baton:start` — the Conductor front porch, slice 1 (PR #71; decisions d061–d065; `1.5.0-rc.1`)
A third front door beside `/baton:go` and `/baton:idea`: an **adaptive, teaching interview**
(name → goal → audience/"done" → folder) → **detect-&-branch folder handling** (adopt or
create+`git init`, every step narrated in plain language) → a **user-owned `CHARTER.md`** in the
project folder → full-auto hand-off to `/baton:go` under its two guards. Resume path reports
plain-language status + the next-command recommendation. Aliases `/baton:init`, `/baton:initialize`.
Box-private `project.json` + `user-profile.json`; depth/teaching precedence: explicit flag >
profile > new-user-defaults > adaptive.

### Slice 2 — working-style learning loop (commit `91864c6`; decision d067; `1.5.0-rc.2`)
The d060 learned-cost loop's sibling, one layer up: an append-only `style-journal.jsonl` observes
each session; a **confidence-gated plurality fold** (≥5 non-explicit sessions at ≥70%) updates the
profile; a Grimdex supplement is **offered, never auto-written** (>85%). Explicit choices always
outrank learned values. Built by the Antigravity/Opus session.

### Slice 3 — mid-stream idea injection (commit `f40a25e`; decision d068; `1.5.0-rc.3`)
Brain-dump-anytime: `--idea "<text>"` (or the resume-path prompt) folds the raw idea into the
CHARTER, then a cheap `triage`-tier classifier routes it — tactical → recommend a `/baton:go`
re-plan; epic-scale → recommend a `/baton:idea` viability debate; **fail-open to re-plan**.
Built by Gemini, committed directly to master without review — see the hardening wave.

### Hardening wave (PR #72; decisions d066 rewritten, d069, d070; `1.5.0-rc.4`)
Post-review hardening of the Gemini-built WIP, which was not deployable as-is:

- **Slice-3 live-path fix:** `Resolve-IdeaRouting` coerced the `Invoke-Fleet` result hashtable to
  a string — classification was silently dead in production, always failing open after spending a
  model call. Also fixed: regex-injection hazard in the CHARTER fold (`$1`/`$$`), hardcoded
  dot-source paths. Suite 7→13 checks.
- **CostResolver (d066):** `Get-RealizedTaskCost` meters exact **cache-aware USD spend**
  (1.25x cache-write / 0.10x cache-read) by cross-referencing run `events.jsonl` task windows with
  local claude-code JSONL logs, wired into `Complete-Run` through the d059 `-CostResolver` seam.
  **CreationTimeUtc attribution** keeps the orchestrator session's own tokens out of task costs;
  unknown models/platforms/logs fall back honestly to the labelled `estimate` basis.
- **GEPA optimizer pilot (d069/d070):** `/baton:optimize-prompt [--max-runs N] [--max-tier T]
  [--apply]` reflects over `polish`/`reject`-gated runs and proposes a mutated Conductor planner
  prompt. **Propose-then-apply, never autonomous:** default writes a candidate file; `--apply`
  validates the `{{schema}}`/`{{evi}}`/`{{Goal}}` placeholders survived, backs up the live prompt,
  then deploys. The planner prompt is externalized (BATON_HOME live copy, **seed-if-absent** on
  deploy → repo copy → embedded fail-open default). Honest scope: v1 is single-shot
  reflect-and-propose, not full GEPA (no candidate population / Pareto frontier).
- Bootstrap deploys all new scripts + seeds the prompt template; both new test suites converted
  to the house Assert/exit-code style.

## Box-private boundary

`project.json`, `user-profile.json`, `style-journal.jsonl`, runs, the live/mutated planner prompt,
and all cost records stay under `$BATON_HOME` and never enter the knowledge repo or any shared
seed. `CHARTER.md` is the deliberate exception — it's the user's own document, in their repo.

## Research base

`knowledge/projects/baton/notes/agent-routing-and-tools.md` — all 11 sources independently
re-verified 2026-07-01 (5 summaries corrected). Key facts: GEPA ("Genetic-Pareto") beats RL with
up to 35× fewer rollouts — the fit argument for Baton's low-run-count environment; DeepEval's
two-model reflection/mutation split + dual acceptance gate is the pilot's graduation path;
Decagon: 20–100 examples optimal, frontier reflection model mandatory.
