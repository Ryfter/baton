## Theme

v1.5 opened the front door and started learning from the user. **v1.6 makes the Conductor's own
brain a population under selection.** The planner prompt is no longer one file that a model
occasionally rewrites — it is a champion defending its title against evaluated challengers, with
lineage, scores, and retirement reasons all on the record. North-star metric (Kevin, 2026-07-01):
**total realized cost to an accepted outcome**, not per-token price — a cheaper prompt or model
that triggers 10× the rework is a net loss, and the pool's bookkeeping is built to prove that
direction in numbers.

## What's new since v1.5.0

### The candidate pool (`scripts/prompt-pool-lib.ps1`)
Box-private state at `$BATON_HOME/prompts/pool/` — a `pool.json` manifest plus one text file per
prompt variant. Every variant carries lineage (`parent`), origin (seed/mutation), status
(champion / candidate / retired-with-reason), offline scores (judge win-rate vs champion, token
estimate, times selected), and **live cost-to-acceptance fields reserved at zero** so the deferred
shadow-A/B slice (Slice B) lands without a migration. Exactly one champion at all times; the pool
seeds itself from the live prompt on first run.

### The GEPA generation loop (`Invoke-PromptEvolution`)
Per `--generations N` (default 1): **select** a parent from the Pareto frontier
(frequency-weighted, DeepEval-style) → **reflect** — a cheap-tier model diagnoses recent
`polish`/`reject`-gated runs (the gate's findings and polish briefs are the natural-language
feedback channel), fed the fates of earlier retired mutations so dead ends aren't repeated →
**mutate** — a stronger model rewrites the prompt → **evaluate** — plan-only head-to-head vs the
champion over those same historical runs, a judge model scoring which plan better answers each
run's recorded findings, position-swapped per example to cancel judge bias, ties excluded →
**dual gate** — the child must strictly beat its parent AND be Pareto-non-dominated on
(judge quality ↑, prompt tokens ↓). Placeholder loss or the 2× length cap retires a mutation
before any evaluation is spent. Retired candidates stay in the pool as reflection fuel.

### Propose-then-apply, still human (d070 unchanged)
Gate survivors are written to `conductor-planner.candidate.txt` for review. `--apply` backs up
the live prompt (timestamped `.bak`), promotes the survivor to champion, retires the old champion
as `superseded`, and marks sibling candidates' scores stale (they re-earn Pareto placement against
the new reference). The Conductor's prompt-resolution chain is untouched.

### CLI + wiring
`--generations N`, `--reflect-tier T` (reflection defaults one tier below `--max-tier`),
`--pool` (per-variant report: lineage, status, scores, live runs). Bootstrap deploys the new
library; the pool itself is runtime state, never seeded. Plugin `1.6.0-rc.1` → `1.6.0`.

## Box-private boundary

and never enters the knowledge repo or any shared seed.
