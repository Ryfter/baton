## What shipped

### `/baton:ship-report` — pipeline-level cost/quality cards (#106)

Per-PR aggregation of what the pipeline actually did and cost, from three sources joined
per PR: the fleet journal (dispatch lines with `tok:N(basis)` telemetry), local git, and
GitHub PR metadata.

- **Card dimensions:** cost (per-stage token totals with the d059 `exact|estimate` basis
  preserved, never silently summed), quality (gate verdicts, review findings, verification),
  choreography (which instruments ran which stages, cap-death/failover events).
- **Stage classifier** prioritizes summary keywords to avoid collisions; **cap-death counter**
  counts lockout events only (no double-counting recoveries); markdown table cells escape
  pipes.
- New: `scripts/ship-report-lib.ps1` (939 lines), `scripts/fleet-ship-report.ps1` runner,
  `commands/ship-report.md`, `scripts/test-ship-report.ps1` (103 hermetic checks).
- This is the **measurement instrument for the #93 bakeoff** — both arms get scored by the
  same cards.

### `--execute` hard-requires task stakes (#101, the d089 flip)

`/baton:go --execute` no longer normalizes missing task stakes with a warning — it
**fails loud**. Every task on the default execute path must carry explicit stakes so the
depth/cost policy is a deliberate, journaled choice (d086: match spend to stakes).

- Review-pass hardening included: plan-invalid worktree/branch cleanup, stderr parity,
  precedence reordering, post-revise invalid-stakes validation.
