## Theme

v1.2 made Baton economic, autonomous, and self-correcting. **v1.3 closes the loop on the
governed fleet**: Baton can now run an *external* model as a self-metering, budgeted worker
inside its routing spine (Sprint 6), and it can render a formal, auditable **accept / polish /
reject** verdict over a finished work artifact via competitive review (Sprint 7). Together these
complete the economic thesis — *spend intelligence like money*: cheap work that is good enough
ships as-is; only work that needs it pays for a premium pass, and a competitive gate decides
which is which. North star unchanged: **autonomy** + **legibility**, with **cost-optimization**
as the through-line.

With v1.3.0 the full v2 MVP is feature-complete: Triage → Usage Governor → Projects → Research
Gate → Conductor → Memory Bridge → Worker Adapter → Acceptance/Polish Gate, all on one
cost-aware routing spine.

## What's new since v1.2.0

### Sprint 6 — Worker Adapter (merged, PR #61 + fix #62; `1.3.0-rc.1`)
Runs `gh models run <model>` as a **budgeted, self-metering fleet worker** — the GitHub Models
allotment becomes a saturable labor pool that reports its own usage into the Usage Governor.
`worker-lib.ps1` (pure parse/meter + seamed dispatch) + `fleet-worker.ps1` CLI + `/baton:worker`.
The seed provider uses **stdin mode** with a valid model id (fix `809551d`) to stay quote-safe and
avoid the pwsh inherited-stdin hang. Hermetic suite + bootstrap deploy asserts; verified live
against the real GitHub Models API.

### Sprint 7 — Acceptance/Polish Gate (merged, PR #63; decision d056; `1.3.0-rc.2`)
The **after-work mirror of the Research Gate**. Research Gate decides *build/adopt/adapt* before
work; the Acceptance Gate decides **accept / polish / reject** after work. `≥2` reviewers review a
finished artifact **independently**; the pure layer normalizes, dedupes (conservative
`area|summary` key), and tags findings **agreed** (raised by ≥2) vs **solo**, then issues a
**severity-driven verdict** (any `critical` → reject; else any `important` → polish; else accept —
thresholds are parameters) plus a ready-to-use **polish brief**.

- **Advisory, not an auto-polish loop (d-ag-1):** on `polish` it emits the verdict + brief; the
  operator or Conductor decides whether to run the premium pass.
- **Independent + deterministic reconcile (d-ag-2):** no adversarial cross-exam round, no third
  "chair" LLM — reconciliation and verdict are pure and deterministic.
- **Strict-JSON reviews over stdin (d-ag-3):** each reviewer emits only `[{severity,area,summary}]`;
  the artifact rides stdin (quote-safe). The seam never parses prose.
- **Fail-open parsing (d-ag-4):** an unparseable review degrades to one flagged "unparsed" review,
  never crashes and never silently changes the verdict; all-unparsed → accept with a flagged reason.

`gate-lib.ps1` (pure: severity rank, parse, merge, verdict, formatters; seamed `Invoke-AcceptanceGate`
with `-Dispatcher`) + `fleet-gate.ps1` CLI (`run`, `--file`/`--diff`/stdin) + `/baton:gate`.
**39-check hermetic suite** + 2 bootstrap deploy asserts. Final Opus whole-branch review:
**READY TO MERGE, 0 Critical / 0 Important.** Live smoke against real reviewers green end-to-end
(both flagged a seeded divide-by-zero `critical` → verdict REJECT).
