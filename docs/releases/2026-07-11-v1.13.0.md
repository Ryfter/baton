# v1.13.0 — Verified Labor V2 (the fleet's work gets checked)

**Date:** 2026-07-11 · **PR:** #84 (squash `b6486a6`) · **Decision:** d082 · **Track:** fleet-does-the-labor / multi-model conductor

V1 shipped the pure verification runner (native, argv-only checks, frozen-from-base-commit
contracts, oracle-integrity grades). **V2 makes `/baton:go` actually use it.** An opt-in
`-Verify` sub-lifecycle runs a task's frozen executable contract after each agentic
attempt, does exactly one evidence-informed retry, and **fails closed** on oracle
tampering. This is the payoff of the Ringer lesson: a labor task passes only when an
independent contract produces durable evidence, and the system remembers whether that
took a retry.

## What shipped

- **Plan normalization** preserves optional `verify_profile` + `allowed_paths` on tasks.
- **Preflight freeze** — before the DAG walk, before any spend: every referenced profile
  is resolved from the base commit and frozen; an unknown / missing / lint-failing
  profile → `plan-invalid` before labor.
- **`New-VerifyingSpawner`** wraps the agentic spawner. Per verified task: freeze the
  pre-task baseline **once**, run the worker, compute the task diff, run the frozen
  contract, enforce `allowed_paths`, apply **outcome precedence** (codex-ringer §7):
  - spawn / infra failure → task fails, NOT a model-quality failure, no retry
  - verification pass despite a nonzero worker exit → stands, warning retained
  - check-fail / timeout / no-change → exactly **one** evidence-informed retry, same worktree
  - scope-violation / protected-oracle mutation → **fail closed, no retry**
  - **A5**: a "pass" over a tree unchanged vs the pre-task baseline is demoted to `no-change`
- **New terminal status `verification-failed`**; six event kinds
  (`task-verification-started/passed/failed`, `task-retry-started`,
  `task-scope-violation`, `task-unverified`); evidence tree
  `tasks/<id>/{contract.json,attempts.jsonl,verification.json,check-output.txt}`;
  a `## Verification` report-narration block.
- Opt-in `-Verify`; **default path byte-for-byte unchanged**.

## Posture

**Fail-CLOSED** — a task that demands verification and can't get it fails (unlike the
advisory gates' fail-open). Verification runs real subprocesses in the worktree; the
evidence lands in the run dir, outside the worker's writable scope.

## Review + hardening

Opus adversarial whole-branch review (trust-boundary + subprocess-in-worktree hostile):
**READY WITH FIXES**, 0 Critical. Four findings fixed:

- **I1 (edge case #4, ruled a defect):** the A5 baseline was re-frozen per attempt,
  judging attempt 2 against attempt-1's end state — false-passing an add-then-revert
  retry (the exact A5 loophole) and false-failing a legitimate no-further-edit repair.
  Now frozen once at task start; VS8/VS9 prove both directions.
- **I2:** the retry evidence prompt could exceed the hard 965-byte inline ceiling,
  silently defeating the retry for inline instruments (agy). Now byte-budgeted ≤965 UTF-8.
- **M1/M2:** `task-verification-started` now emitted (six-kind contract literal); retry
  spend summed.

## Live proof (2026-07-11)

Real codex labor into a worktree + a real committed `pwsh` check, frozen from the base
commit, run as a subprocess. The **failure path was proven first** (verification caught
absent work → one retry → `verification-failed` with the full evidence trail), then the
**pass path** (codex created `greeting.txt`="hello world" → check verified → PASS, grade
bounded, first try → `completed`, effective_cost 0.05 measured). A live-smoke bug was
caught and fixed en route: an empty task desc crashed the retry via a mandatory
`[string]` param (`[AllowEmptyString()]` + VS7 regression, `cc1423c`).
