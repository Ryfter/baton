# v1.8.1 — coach-footer deploy fix

**Date:** 2026-07-04 · **Fix:** `7ee66c2` · Patch on v1.8.0 "The Coach".

A one-line deploy-manifest omission meant half of v1.8.0 never ran in
production. This patch fixes it.

## The bug

`scripts/coach-lib.ps1` was missing from bootstrap's Step 5b deploy
manifest, so `~/.claude/scripts/coach-lib.ps1` never existed on deployed
boxes. The fleet CLIs (`fleet-usage`, `fleet-gate`, `fleet-go`,
`fleet-optimize-prompt`) dot-source it fail-open (`try { . } catch { }`),
so `Write-CoachFooter` was undefined and the **`Next:` footers silently
no-op'd on every deployed box** since v1.8.0.

The SessionStart **digest was unaffected** — the `baton-coach.ps1` hook
resolves `coach-lib.ps1` from the plugin root, not the deployed scripts
dir. That is why the v1.8.0 live smoke looked green: it exercised the
digest (which worked), never a footer (which didn't).

## The fix

- `coach-lib.ps1` added to the Step 5b deploy manifest in
  `scripts/bootstrap.ps1`.
- `scripts/test-bootstrap.ps1` gains an assert so the manifest can't drop
  it again (55/55 green).

Manifest-only; no behavior change to any library. Deployed boxes pick up
the footers on the next `bootstrap.ps1 -Force`.
