# v1.12.0 — The Plan Gate (Claude conducts; Codex + Grok once-over)

**Date:** 2026-07-10 · **PR:** #82 (squash `8fac9d1`) · **Decision:** d080 (+ d081) · **Track:** multi-model conductor

The third lifecycle gate. Research Gate (d050) decides *before building*; Acceptance
Gate (d056) judges *after work*; the **Plan Gate** critiques the plan *after planning,
before any labor spend* — peer fleet models independently review the Conductor's task
DAG so Claude never grades his own homework.

## What shipped

- **`grok-cli` joins the fleet** (Slice 0): first-class provider, `platform: grok`,
  `agentic: true`, new `plan-review` capability; codex granted explicit capabilities
  (general trio restated + `plan-review` + `review`).
- **`{{prompt_file}}` transport** (d081): third prompt path — prompt written to a temp
  file, path substituted as a single argv token (no `Invoke-Expression` reparse),
  `finally`-deleted. For CLIs that neither read stdin nor tolerate inline quotes (grok).
- **`plan-gate-lib.ps1` + `/baton:plan-gate`** (Slice 1): independent JSON findings →
  deterministic merge → **accept / revise / reject**; understaffed roster (<2, post-
  dedupe) fail-opens; CLI exits 0/1/2 (accept / revise-reject / user error).
- **Conductor seam** (Slice 2): opt-in `-PlanGate` on `/baton:go` — gate runs after the
  plan parses, before the walk; `reject` → terminal `plan-rejected` with zero labor
  spend (and no leftover worktree under `-Execute`); `revise` → ONE auto-revise
  (`-PlanRevise:$false` to skip), fail-open to the original plan; gate-infrastructure
  failure never kills a run. New run artifacts: `plan-review.json`, `revise_brief.md`.
- **gate-lib parse hardening** (shared with the Acceptance Gate): balanced-array
  scanning last-first + echoed-schema reject + shape validation + unmatched-`[`
  resilience — prompt-echoing reviewers (codex) now parse.

## Live proof (2026-07-10)

- Deliberately flawed plan → standalone gate: **REJECT**, both reviewers contributing
  (codex 3 findings + grok 6), revise brief aggregating both.
- `/baton:go -PlanGate` conductor run: **grok authored the plan** (a first), the gate

  status `completed`.
- Real-outage resilience: codex hit its usage limit mid-campaign → gate degraded it to
  unparsed and proceeded — the fail-open posture proven against a live provider outage.
