# v1.19.0 — The instrument ABI: any language, one contract (d091, #92 + #104)

**Released:** 2026-07-17 · **PRs:** #109 (instrument ABI, `7368fe4`) + #110 (context_overflow,
`123460c`) · **Issues:** #92, #104 closed · **Decisions:** d091 (transports behind one
return contract) · **Spec:** `docs/superpowers/specs/2026-07-17-instrument-abi-design.md`

The "add any instrument" thesis now holds structurally: a fleet worker is anything that can
produce one honest row — `{ stdout, stderr, exit_code, duration_s, tokens, tokens_basis }` —
and three transports produce it.

## Instrument ABI (#92, d086 spine node #4)

- **Generic OpenAI-compatible HTTP invoker** for `kind: http` fleet rows. The three
  copy-paste per-provider hatches (lm-studio, lm-studio-small, ollama-box2) are deleted;
  their rows ride the generic path and now report **exact** token counts from the server's
  `usage` block (was `estimate`). A per-provider `scripts/fleet/<name>.ps1` hatch still
  wins when present — the escape valve stays. Model resolution preserves d043.
- **tools.yaml routability:** `kind: python` and `kind: http` instruments dispatch; the
  cli-only gate widens to a supported-kinds set; unknown kinds still skip loudly.
- **`stdio-json` transport:** any executable speaking one JSON request on stdin / one JSON
  response on stdout — any language, zero Baton changes to add an instrument. Request rides
  a temp file (965-byte rule).
- **Declaration fields** (additive): `max_prompt_bytes` (enforced pre-dispatch as UTF-8
  bytes), `probe` (d090 slot), `agentic` — with `agentic: true` structurally CLI-only (d009).

## context_overflow (#104, the reactive half)

- New classification for the local-model silent death (2026-07-16 incident: 80KB prompt →
  exit 1, empty output, indistinguishable from a dead server). Signals: overflow strings
  (quota-phrased token limits deliberately excluded) or nonzero exit + empty output +
  prompt ≥ floor (35KB default; `max_prompt_bytes` overrides).
- **No lockout, no cooldown** — the provider is healthy; the remedy is split or reroute.
  Operator line: `prompt too large for <provider> (<n>KB > <cap>KB) — split the prompt or
  reroute to a larger-context peer` (`?KB` when size unknown).
- One quality_first substitute hop, soft-preferring a larger-context peer. Accepted risk
  (named in code): a dead provider + empty output + large prompt gets no cooldown; the
  6-token probe stays the tiebreaker.

## How it was built (the fleet story)

Codex started #92 from the spec and was **halted ~5 minutes in** when Kevin reserved his
weekly Codex quota — the d090 machinery itself enforces the hold (`soft_cap_weekly` set
below current usage; pre-flight refuses dispatch until the reset). The 5-minute snapshot
turned out essentially complete; **Grok reconciled it against the spec and verified
end-to-end**, then built #104 outright. **Opus served as adversarial reviewer** (the
implementer never reviews its own work): SHIP-AS-IS on #92 (3 nits), SHIP-WITH-TWEAKS on
#104 (5 findings; the real one — overflow regexes broad enough to swallow token-metered
quota messages — fixed with regression tests both ways). 36 local-lens tripwire findings
across both PRs: 33 refuted, 3 folded into fixes.

**Dogfood:** the review's own local lenses ran through the new generic HTTP invoker —
journal telemetry flipped from `tok:N(estimate)` to `tok:N(exact)` live, mid-pipeline.
Exact local token counts are the prerequisite for an honest #93 bakeoff.
