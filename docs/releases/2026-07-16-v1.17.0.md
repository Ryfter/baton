# v1.17.0 — Execute governance goes authoritative (d086 node #1)

**Date:** 2026-07-16 · **Track:** feature (`scripts/fleet-go.ps1`, `scripts/conductor-lib.ps1`, `scripts/fleet-executor-lib.ps1`, `scripts/plan-gate-lib.ps1`, `scripts/gate-lib.ps1`) · **Follows:** v1.16.0 · **Decisions:** d086, d087, d088

`/baton:go --execute` used to run its governance as advisory theatre: gates defaulted off,
and a degraded gate quietly accepted. This release flips the golden path — governance is
**default-on and fail-loud** — and adds the first **depth-matched-to-stakes** routing, so
high-stakes tasks get champion-tier scrutiny while low-stakes ones stay cheap. Shipped as two
squash-merged PRs per the d087 split: PR-A #99 (gates flip) and PR-B #100 (stakes routing).

## What's new

### PR-A — gates defaults-flip + fail-loud (#89)

- **Default-on governance under `--execute`** — Plan Gate, the named acceptance panel
  (`--panel --fail-loud`), and verification all run by default. Escapes: `--no-plan-gate`,
  `--no-gate`, `--no-verify` — each restores only its own legacy node; `--no-gate` still
  writes `changes.diff` (proof-by-diff survives).
- **Degraded gates HALT** — new non-success execute statuses `plan-gate-degraded`,
  `acceptance-degraded`, and `needs-polish` exit 1 with the report emitted first;
  infra-degraded plan gates get the same worktree/branch cleanup as plan rejection.
  Infra-`degraded` stays distinct from content verdicts.
- **Non-execute paths byte-for-byte unchanged**; `Invoke-Conductor` library semantics
  preserved for existing callers.

### PR-B — depth matched to stakes (#98)

- **Planner stakes schema** — every planned task carries `stakes` (`low|standard|high`) and
  `stakes_basis`; `--stakes <level>` (`-StakesOverride`) lets the operator override the whole
  run, recorded as `operator override` basis.
- **`Resolve-TaskDepthPolicy`** — the locked mapping: low → cheap/economy under free caps;
  standard → medium depth, economy; high → champion selection at the run's ceiling.
- **Provider depth tiers** — CLI providers with named tier fragments (`tier_low/med/high` +
  `{{tier_args}}`) get the matching effort tier; `depth_applied` is a truthful capability
  flag, and unsafe fragments are refused.
- **Legible decisions** — `decisions.jsonl` gains additive fields (`stakes`, `stakes_basis`,
  `depth_tier`, `depth_applied`, `selection_mode`, `tier_cap`, `selected_cost_tier`);
  `cost_tier` still means the planner estimate. The run report narrates the policy per task.
- **d088 cleanup** — the dead `-RequireVerify` Conductor parameter is dropped; enforcement
  lives in fleet-go's execute preflight, where it always did.
- Missing stakes on execute stays **normalize+warn** for now (ship-soft); the hard-require
  flip is tracked as #101.

## How it was built

The most fleet-built release yet, across a Codex usage-cap interruption: **Codex built**
(both PRs, resuming mid-build from a WIP commit after its cap reset), **Grok adversarially
reviewed** (SHIP-AS-IS on PR-A; SHIP-WITH-TWEAKS on PR-B, then implemented its own four
tweaks), **two local models** (lm-studio ×2) ran free correctness/test-hygiene lenses, and
Claude conducted: scheduled the resume for the cap reset, cross-checked every lens finding,
and independently re-ran the full gate before recommending merge.

## Verification

defaults flip, each escape's node-locality, degraded-halt exits, the operator stakes
override end-to-end into `decisions.jsonl`, and positional-ABI compatibility for existing
spawner callers.
