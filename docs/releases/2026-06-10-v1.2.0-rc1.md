## Theme

v1.1.0 closed out Plans 1–11 and cleared the backlog. v1.2.0 is the **Fleet Conductor**
evolution: the orchestrator grows from a single-session job runner into a conductor that
routes work across a fleet of models + tools, learns which to use, and shows you what it's
doing. North star: **autonomy** (stop forcing 1/2 prompts) + **legibility** (always show, in
plain English, what each agent is doing) — with **cost-optimization** as the through-line.

## Highlights

- **Capability-routing optimizer (S1→S4)** — the flagship. An explainable, cheapest-tier-first
  auto-router over `tools.yaml` + `fleet.yaml` that dispatches, verifies, escalates up the
  cost ladder, **learns** from ratings + an LLM judge, and supports a **calibration** mode.
- **Cost-Optimization Engine — Slice A (time-awareness)** — rank-gates paid/frontier dispatch
  during prime-peak hours and scales concurrency up during off-peak/weekend surge.
- **Legibility dashboard + coordination backbone (SP2)** — a file-based runs feed and a FastAPI
  dashboard; real dispatched fleet agents surface in an assignment view.
- **`/idea` front door (SP3)** — one command turns a raw idea into board-ready GitHub issues.
- **Tools registry + Docling (d025)** — a non-LLM capability registry, co-equal sibling of the
  model fleet.
- **Grimdex** — the knowledge base spun out into a standalone, tool-agnostic product; this repo
  is wired to it with graceful degradation (works with or without it).

## What shipped (since v1.1.0)

### Capability-routing optimizer — auto-router + learning loop (decision d026)
The cost-optimization spine: orchestrate cheap/local models + tools, pick the *optimal*
(not most-powerful) capability, and get smarter over time.

- **S1 — selector + data model** (merged `82be1b9`). `Select-Capability` (`routing-lib.ps1`)
  returns a ranked, explainable, cheapest-tier-first candidate list (`local`<`free`<`paid`);
  `/route <capability>` shows the pick + why. Specialty models migrated into `tools.yaml`.
- **S2 — auto-dispatch + verify/escalate** (merged `39c63b2`, closes #34). `Invoke-RoutedCapability`
  (`routing-dispatch.ps1`) walks the cost ladder, dispatches each candidate, grades it, and
  escalates on failure → terminal `escalate-to-conductor`. Every attempt logged to
  `routing-journal.jsonl`. Pluggable `-Grader` seam (d027).
- **S3 — ratings + learning loop** (merged `2656ce4`, closes #35). `Get-CapabilityQuality`
  (`routing-learn.ps1`) blends user ratings + LLM-judge + heuristic history into a learned
  within-tier tiebreaker (cost still dominates). `/route --rate good|bad`; ratings persist to
  the GitHub-backed store. Decisions d028/d029/d030.
- **S4 — calibration mode** (merged `b88b12b`, closes #36). `/route --calibrate` fans out across
  all candidates, judge-scores each, and shows a side-by-side table with learned-quality
  provenance + a pre-filled rate command. The exploration twin of S3's exploitation. Decision d031.

### Cost-Optimization Engine — Slice A: time-awareness (decision d026, levers)
(merged `e584e7b`). New pure lib `prime-hours.ps1` — `Test-PrimeHoursGate` rank-gates paid
dispatch inside `peak` windows (rank 1=ask/run … 5=defer; reserved 0/6; fail-open), and
`Get-CapacityProfile` reports a weekend/off-peak concurrency surge. Wiring is opt-in and
regression-safe: routing paid-tier `-Rank` gate, backlog effective-rank prereq inheritance +
per-item gate, `/route --rank`. **Invariant: rank ≠ tier** — the gate only governs
premium-spend-*now*; the optimizer still picks the cheapest capable model.

### Fleet Conductor surface — legibility + coordination
- **Legibility dashboard (Slice 1)** (merged `0c0f274`). A file-based feed under `~/.claude/runs/`
  written by PowerShell hooks and read by a FastAPI dashboard: runs gutter + detail pane +
  needs-you answer queue. Decisions d018 (conductor not monolith), d019 (web is the primary surface).
- **Dashboard redesign** (Gemini, merged 2026-06-05) — zero external dependencies (vendored
  htmx/Chart.js, system fonts) so it renders fully offline.
- **SP2 — coordination backbone** (merged `5027956`). A fleet→runs bridge wires real dispatched
  agents into the feed; a per-agent assignment view with active/queued/parked-for-human lanes;
  current-run wiring so the conductor's own session narrates in. Decisions d020/d022.
- **SP3 — `/idea` front door** (merged `b348855`). KB prefetch → research ensemble → council
  viability debate → concept doc → GitHub issues, with one human gate. Decision d023.

### Tools registry + Docling (decisions d024/d025)
(merged `5573ecc`). `tools.yaml` — a non-LLM capability registry, co-equal sibling of
`fleet.yaml`; tools declare `cost_tier` + `capability`, invoked per `kind` (python/cli/http).
First entry: Docling (`pdf-extract`, local), wired into KB ingest with a lazy/optional import
so a PDF is never silently zero-chunked.

### Grimdex — knowledge base spun out (decisions d032/d033/d034)
The KB became a standalone, tool-agnostic product (`GRIMDEX.md` root convention). This repo is
wired with `<!-- grimdex:start -->` pointer stanzas and a graceful-degradation contract: it
detects the `GRIMDEX.md` marker and lights up; falls back to plain local markdown when absent.
Mutual independence — neither depends on the other. The three global `~/.claude/rules` are now
mirrored + redeployable, closing a backup gap.
