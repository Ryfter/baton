# v1.10.0 — Fleet Round-Trip Proof (Slice 1 of "the fleet does the labor")

**Date:** 2026-07-05 · **Merge:** `9602c43` (PR #79) · Minor on v1.9.0.

Baton's thesis is command-and-control for a fleet of coding LLMs — but until now
that was only proven with Claude. This slice proves the pipe: it confirms each
enabled instrument actually **answers a real prompt**, and hardens the dispatch
path so the real prompts a later slice will send survive. It does **not** apply
repo changes — the executor that turns a model's output into a committed change
is Slice 2.

## What's new

- **`fleet doctor --live`** — a real round-trip probe. It sends a **4-challenge
  canary battery** to every enabled provider (CLI and HTTP) in a single dispatch
  and reports, per provider, `live_ok | live_fail(reason) | skip`:
  1. *Reply with the word PONG.* → `PONG`
  2. *What is 6 times 7?* → `42`
  3. *What is the capital of France?* → `PARIS`
  4. *What is the opposite of HOT?* → `COLD`

  The response is scored k/4 (case-insensitive). Only `PONG`'s answer appears in
  the prompt, so a provider that merely **echoes** the prompt (a wrong template)
  scores 1/4 and correctly fails — the payoff line is a `no-canary` / partial
  reason that tells you the template is wrong for this box, versus a model that's
  simply down. `--timeout <s>` (default 60) and `--json` are supported; exit 0
  iff every enabled provider is `live_ok`, else 1. The default (non-live) `fleet
  doctor` is unchanged.

- **Prompt hardening.** CLI templates ending in a standalone quoted `"{{prompt}}"`
  (`codex exec`, `claude -p`, `agy --print`, `gh copilot suggest`, `ollama run`)
  now pass the prompt via **stdin** instead of interpolating it — immune to the
  965-byte argument ceiling and quote mangling. Embedded-`{{prompt}}` and
  shell-wrapped templates keep the legacy interpolation path.

## How it works

- A new `scripts/fleet-probe-lib.ps1` holds the pure canary classifier
  (`Test-FleetCanary`), reachability check (`Test-ProviderReachable`), and the
  timeout-guarded live probe (`Invoke-FleetProbe`, using `Start-ThreadJob`).
- Model-agnostic by construction: the probe dispatches through the existing
  harness-neutral `Invoke-Fleet`, so any provider any agent adds is probed the
  same way.

## Quality

fleet-doctor 12, bootstrap 59 (+1 deploy assert), conductor 105 (no regression).
Opus whole-branch review: **READY TO MERGE, 0 Critical**. Two fixes landed after
review: the canary was upgraded from a single `PONG` token to the 4-challenge
battery (the token had lived inside its own prompt, so an echo false-passed); and
a **stdin:true dispatch regression** was caught and fixed — the stdin hardening
had routed `stdin:true` providers through a mandatory-parameter rejection,
breaking real providers (github-models, box-private reviewer pairs). A
`stub-stdin` fixture + round-trip asserts now cover that path.

## Upgrade note

Because the stdin promotion changes how the prompt reaches trailing-quoted
providers, **re-run `fleet doctor --live` after upgrading** to confirm your box's
roster still answers. A provider that doesn't read stdin will show up as
`no-canary` until its template is adjusted.
