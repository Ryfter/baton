## Fixes

1. **Planner parse for prompt-echoing providers** — codex echoes the prompt (incl.
   the plan JSON schema) before its answer; greedy `{`…`}` extraction spanned
   echo+answer → `plan-failed`. `ConvertTo-PlanObject` now tries string-aware
   balanced JSON blocks LAST-first after the greedy fast path.
2. **Echoed-schema guard** — an echo-only reply can't be mistaken for a plan (the
   schema placeholder `est_cost_tier: "local|free|paid"` is the reject signature).
3. **`stdin: false` promotion veto** (fleet.yaml field) — agy neither reads stdin
   (bare `agy --print` = "flag needs an argument") nor honors a `-` sentinel (treated
   as an EMPTY prompt: exit-0 generic greeting). `stdin: false` keeps a provider on
   inline interpolation; caveat documented (quote-free prompts only for agy).
4. **Planner capability vocabulary** — the code-side schema now enumerates the
   canonical routing capabilities and forbids baton command names (codex had planned
   capability `"code-parallel"` → no candidate → walk failed).
5. **codex `--sandbox workspace-write`** (seed template) — untrusted dirs (throwaway
   run worktrees) downgrade codex to a read-only sandbox; with `approval: never` it
   exits 0 having edited nothing.
