# v1.14.0 — Copilot Credit Budget (the fleet's spend gets a window)

**Date:** 2026-07-11 · **PR:** #85 (squash `bffee48`) · **Decision:** d079 · **Track:** guided-use / cost-visibility

`/baton:usage` now surfaces a **fail-open advisory panel** for GitHub Copilot AI
credits, read live from the GitHub billing API and forecast against the billing
cycle. Born from the 2026-07-06 surprise — 1018/1500 monthly credits consumed with
no visibility (culprit: automatic Copilot PR code review, since disabled). This gives
the number a window before it becomes a bill.

## What shipped

- **`scripts/copilot-credit-lib.ps1`** — fetches
  `users/<user>/settings/billing/ai_credit/usage` (needs `gh` `user` scope) behind an
  injectable fetcher seam. Config reuses the Governor `budget` field plus new
  box-private `credit_reset_day` / `credit_warn_pct`. Cycle-anchored run-rate forecast
  with honest statuses (`unavailable` / `no_budget` / `no_reset_anchor` / `ok`).
- **`/baton:usage` panel** — appears only when a budget is configured; `--json` gains a
  `copilot_credits` block on the same condition. Local budget read gates *before* any
  network fetch, so the no-config path never touches GitHub.
- **Fail-OPEN everywhere** — a missing scope, a 403, an unreachable API, or an
  unconfigured budget degrades to a one-line hint (e.g. `gh auth refresh -s user`),
  never an error. Advisory only; nothing throttles.
- Deploy wiring (bootstrap manifest + assert), command doc, box-private seed
  placeholders, AGENTS.md line, plugin minor bump.

## Posture

**Fail-OPEN** — this is a visibility aid, not a gate. Per-source credit split is
impossible on personal GitHub accounts, so the per-model total is the ceiling the
panel reports. Estimates are never presented as metered (the d059 honesty rule).

## Review + hardening

Opus whole-branch review: **READY WITH FIXES**, 0 Critical. Fix wave (`4cb2770`):

- **#1 (Important):** the panel used `·` (U+00B7) despite its own ASCII-only contract —
  mojibake on a non-UTF-8 Windows console. Replaced with ASCII `|`; P1b assert added.
- **#2 (hardening):** a 403 / "not accessible" response wasn't classified as
  insufficient-scope, so the actionable hint was withheld. Now matched; C3b assert added.
