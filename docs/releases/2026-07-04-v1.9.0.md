# v1.9.0 — The Project Command Center (Layer 1)

**Date:** 2026-07-04 · **Merge:** `a201668` (PR #78) · Minor on v1.8.2.

Baton becomes the single front door for every project. Launch it from a
`D:\dev` home base and drive any project by name — no `cd` first.

## What's new

- **Run a project by name from the home base.** `/baton:go --whimsicalcarving
  <goal>` resolves the name to that project's folder (`D:\dev\WhimsicalCarving`)
  and runs the existing Conductor there. `--<slug>` is shorthand for
  `--project <slug>`. From inside a project folder, `/baton:go <goal>` targets
  that folder as before; from `D:\dev` itself, use the picker.
- **A live project roster.** `/baton:project list` shows every project grouped
  **Active** (a session is open in the folder) / **Inactive** (registered, no
  open session — may be `[resumable]`) / **Archived** (done with it). Plus
  `archive` / `unarchive` / `hide` / `set-blurb`.
- **Resume where you left off.** When a session ends, Baton records an
  agent-tagged resume pointer; the CLI surfaces `claude --resume <id>` for an
  inactive-but-resumable project.

## How it works

- A **scan-seeded hybrid registry**: Baton scans `D:\dev` (a folder counts if
  it has a `.git` dir or a `CHARTER.md`) and folds the scan over the existing
  per-project `project.json` records — no duplicate store. The registry owns
  blurbs, `archived`, and `hidden`; the scan owns discovery. Override the home
  base with `$BATON_HOME`-sibling `$env:BATON_PROJECTS_ROOT`.
- **Active/inactive is detected, not declared** — from a neutral session-marker
  contract under `$BATON_HOME/sessions/` (`{agent,session_id,cwd,started_at}`),
  written by SessionStart/SessionEnd hooks, aging out fail-open.
- **Model-agnostic by construction.** The core (registry, marker contract,
  resume record) is harness-neutral PowerShell; only the two hooks are
  Claude-specific. The same marker/resume contract lets Codex plug in — the
  Codex front door is documented in `AGENTS.md` as a follow-on.
