# v1.7.0 — The Proving Ground release (stable)

**Tag:** `v1.7.0` · **Date:** 2026-07-02 · **Previous:** `v1.6.0` (The Evolving Baton), rc line: `1.7.0-rc.1` (optimizer graduation Slice B)

Promotes `1.7.0-rc.1` to final. One feature, deep: the GEPA optimizer's loop closes with **live
evidence** — challenger prompts prove themselves in dollars on real `/baton:go` runs, and the pool
answers promote/retire on the metric that actually matters. (PR #74; spec
`2026-07-02-optimizer-slice-b-live-shadow-ab-design.md`; decision d072.)

---

## Theme

v1.6 made the Conductor's planner prompt a population under selection — but selection ran on an
offline judge's opinion. **v1.7 makes real work the judge.** A candidate that survives the offline
dual gate becomes the *challenger*: live runs alternate champion/challenger, every run's
CostResolver-metered realized cost and acceptance verdict accrue to the variant that planned it,
and the decision figure is **cost per accepted outcome** (`realized_cost_usd / accept`) — Kevin's
north star, with rework dollars in the numerator by construction. A prompt that is cheaper per run
but triggers more rework loses on the number that counts.

## What's new since v1.6.0

### Live shadow A/B (`Invoke-PlanPhase` + `Complete-Run`)
`Resolve-ShadowVariant` assigns each `/baton:go` run to whichever of champion/challenger has fewer
live runs (tie → challenger; self-balancing across aborted runs). Challenger runs plan through the
pool's candidate text via a placeholder-validated `-Template` override; champion runs are
bit-identical with before. The assignment is legible: a `shadow` event in the run log +
`shadow.json` in the run dir. After the report renders, realized cost accrues to the variant's
`live.*` fields — every `polish`/`reject` dollar also counts as rework. **Fail-open everywhere**:
no pool, no challenger, `--shadow off`, unreadable challenger text, or an accrual fault → the run
behaves exactly as before Slice B, at most one warn event.

### Auto-retire losers; promotion stays human (d070)
At ≥5 gated runs per variant, a challenger losing on cost-per-accept is **retired automatically**
— it stops burning money on real runs. A winning challenger earns a `PROMOTE` recommendation in
the run log and the pool report; deploying it still takes the human `--apply`. Baton acts alone
only in the direction that stops spending.

### One retirement door, full provenance
Every retirement — mechanical rejection, dual-gate failure, superseded champion, live A/B loss —
now flows through `Set-CandidateRetired` and records **why** (reason with the dollars), **when**
(`retired_at`, UTC), and **what beat it** (`retired_by`). Schema additions are additive
(still schema 1, zero migration).

### CLI + kill switch
`--shadow on|off` (persisted in `pool.json`; absent = on — shadow is on by default once a gate
survivor exists). `--pool` grows live columns (`runs/acc/pol/rej/real$/rework$/$per-accept`) and a
shadow-verdict footer (`insufficient (n/5)` · `PROMOTE pNNN — run --apply` · `stalemate`).
Plugin `1.7.0-rc.1` → `1.7.0`.
