## M1 — routed promotion writes into the Grimdex tree

`Write-PromotionToGrimdex` is now a router. A promotion's scope and kind
decide which shelf of the real Grimdex layout it lands on:

- `universal/mistakes.md` (avoid-kind), `universal/winners.md` (prefer-kind),
  `projects/<id>/decision-guidance.md` (project-scoped), or box-private
  fallback.
- Scope is derived from the rows — any `scope=universal` row wins.
- Root resolved from `BATON_GRIMDEX_ROOT`; **fail-open** to a box-private
  path with a `Write-Warning` if the tree is absent, so a promotion is never
  lost.
- Each write carries a dated provenance header
  (`<!-- baton memory-promote <date> | signature: <sig> | source: memory-journal -->`)
  and is append-only.

## M2 — order-stable atomic promotion stamp (+ latent data-loss fix)

`Set-MemoryPromoted` was rewritten line-wise:

- Untouched and **malformed** journal lines are preserved byte-verbatim. The
  old code silently **dropped** malformed lines on every rewrite — a latent
  data-loss bug, now fixed.
- Touched rows round-trip through an `[ordered]` map so field order is
  stable; ISO dates are re-stringified to `'o'` (the `ConvertFrom-Json`
  DateTime auto-parse trap).
- Writes are atomic: a random-named temp file + `Move-Item -Force`.
